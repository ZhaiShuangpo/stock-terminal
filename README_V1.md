# 📈 大A盯盘终端 (Stock Terminal) V1.0 发布说明与技术规格文档

## 1. 项目概述
**大A盯盘终端 V1.0** 是一款专为 A 股投资者设计的现代化、高信息密度 Web 盯盘工具。系统采用“后端行情引擎 + 前端极客交互”的架构，旨在解决传统行情软件信息零散、缺乏即时 AI 逻辑归因以及 Web 端交互卡顿的痛点。

---

## 2. 核心功能特性

### 2.1 全景实时监控 (Market Dashboard)
- **毫秒级推送**：基于 WebSocket 实现行情数据流式推送，界面无刷新平滑更新。
- **高密度列表**：支持自选股自定义拖拽排序（Dnd-kit 驱动），严格遵循 A 股红绿视觉习惯。
- **多维指标**：涵盖现价、涨跌幅、成交量、成交额及 Sparkline 迷你分时趋势图。

### 2.2 极客交互与隐私保护
- **Boss Mode (老板键)**：按 `Esc` 键一键切换至“系统监控”伪装模式。界面转为黑白灰调，隐藏红绿色彩，并将股票名称伪装成系统进程 ID（如 `SVC-0519`），适配办公场景。
- **本地化存储**：自选股分组、API Key 等敏感配置完全存储在浏览器 `localStorage` 中，不经过中心化服务器，保障隐私安全。

### 2.3 深度 AI 投研辅助 (Gemini 2.5 + Search)
- **个股 AI 极速推演**：结合 **Google Search 联网能力**，实时抓取当日新闻、公告。
- **实战 Prompt 驱动**：从“资金定性”、“核心逻辑”、“关键点位”、“操作剧本”四个维度提供游资级操盘建议。
- **智能盘后复盘**：一键生成全市场复盘报告。V1.0 版本已支持**逐一诊断自选股**，并根据实时涨跌幅与全网消息给出次日沙盘推演。

### 2.4 可视化分时交互
- **专业级图表**：集成 Lightweight Charts，支持秒级分时走势。
- **悬浮动态数据板**：鼠标悬浮分时线时，实时显示该点位的**具体价格、时间及相对于昨日收盘的动态涨跌幅**。

## 3. 技术架构设计

### 3.1 后端行情引擎 (`/backend`)
- **核心框架**：Python FastAPI (异步非阻塞架构)。
- **行情适配层**：
  - `fetch_tencent_data`: 批量获取实时行情与异动检测。
  - `fetch_indices`: 指数（上证、深成、创指）同步抓取。
  - `fetch_sectors`: 领涨行业板块抓取。
- **AI 交互层**：集成 `google-genai`，开启 `tools=[{"google_search": {}}]` 联网搜索增强，消除大模型幻觉。
- **双工通信**：WebSocket 维持长连接，支持前端动态订阅（`subscribe`）不同自选池。

### 3.2 前端交互界面 (`/frontend`)
- **核心框架**：React 19 + TypeScript + Vite 8。
- **状态管理**：原生 `useState` + `useRef` 优化高性能 WebSocket 消息处理。
- **样式方案**：Tailwind CSS 4.0，支持深度定制的暗黑皮肤。
- **关键组件**：
  - `Chart.tsx`: 封装 Lightweight Charts，处理分时数据渲染与 Tooltip 逻辑。
  - `App.tsx`: 全局业务逻辑路由、Dnd 排序及搜索分发。

## 4. 开发、测试与维护指南

### 4.1 环境准备
- **后端**：Python 3.12+，需安装 `requirements.txt` 中的依赖。
- **前端**：Node.js 18+，使用 `npm install` 安装依赖。

### 4.2 运行指令
- **一键启动**：根目录下执行 `sh start_all.sh`。
- **手动调试**：
  - 后端：`cd backend && source venv/bin/activate && python main.py`
  - 前端：`cd frontend && npm run dev`

### 4.3 自动化测试
- **WebSocket 连通性测试**：执行 `python backend/test_ws.py`。该脚本会自动连接并验证前 2 条行情推送是否符合预期字段格式。
- **前端构建检查**：执行 `npm run build` 验证 TypeScript 类型安全与生产环境分包。

### 4.4 后续维护建议
1. **行情接口扩展**：若腾讯接口频率受限，可在 `main.py` 的 `fetch_tencent_data` 中加入对新浪或东财 API 的容灾切换逻辑。
2. **AI 模型升级**：当前默认使用 `gemini-2.5-flash` 以平衡速度与成本，如需更高精度分析可修改 `model` 参数。
3. **性能优化**：当自选股超过 100 只时，建议在前端引入虚拟列表（Virtual List）渲染。
